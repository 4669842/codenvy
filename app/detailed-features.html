Primary Workbench

Thousands of developers have made Codenvy their workbench of choice. Whether working alone or as part of a team, Codenvy lets you build any type of application, on any device, anywhere in the world.

Commuter

Whether commuting to work or checking a build during vacation, your configuration is always ready. Move seamlessly between different devices that you have access to. Codenvy brings your configuration with you everywhere.

Chromebook Development

Developers with Chromebooks use Codenvy as an auxiliary environment. It loads quickly and can be synchronized with your code repository to make moving between machines easy. And if you are on a Chromebook Pixel, the image is just stunning.

Hackathons

Planning to hack something together with friends? Only have a limited window to get hacking? Codenvy brings teams together, whether spontaneously or over a longer timeframe. Codenvy automates project creation and sharing with simple URL formats to let you spend more time hacking and less time trying to hack.

Classrooms

Codenvy can be used in a Webex-style collaboration mode ideal for instructors broadcasting code to groups all at once. Also, our Factory makes it easy to create exercises cloned by each student without having to deal with configuration instructions.

Pair Programming

Experience the extreme productivity gains of programming together with others. Codenvy allows developers to edit the same code, chat together in real-time, and even monitor each other’s builds and debugging sessions.

A Development Platform

SDK

Our Eclipse-like SDK enables you to change the programming language, frameworks, build system, runtime, and feel of the system. We also provide a downloadable version of Codenvy that can be used to test plug-ins outside of the Codenvy cloud.

RESTful APIs

Codenvy is a service-oriented platform. Every function is first implemented as a RESTful Web Service and then the UI is layered on top. You can write macros, applications, and scripts that use these APIs to access and control Codenvy.

Codenvy Enterprise

Code, build and test, in a private on-premises cloud. Bring your own hardware and scale up development to any size. You install and manage Codenvy behind your firewall, control access to your code, and provision workspaces to global teams.

Connects to Multiple Systems

Codenvy can connect to cloud datasources, code management repositories, maven central, and PaaS systems. Connect to PAAS like Google App Engine, Amazon Web Services BeanStalk, Heroku, Tier3, AppFog, CloudFoundry, OpenShift, and CloudBees. Connect to cloud datasources like Postgres, MySQL, Oracle, and MS SQL.

Analytics and Data Feeds

All code, build, and test activity is logged. You can track user logins and monitor who has access to your code. Group analytics are also available to track invitations, access, engagement, and virality of your work.

Workspace, Project, and Code Management

Virtual Workspace

Your workspace is virtualized across a set of IDEs, projects, code repositories, builders, and runners. Use the workspace just for yourself or invite others. Files and configuration are kept synchronized across all systems.

Workspace Scaling

Your workspace can have 1 or more queues allocated to builders and runners ensuring that each workspace can scale to the demands of the application you are building. Free accounts use shared queues and premium accounts have dedicated queues.

Shell

We provide a browser-based shell that lets you invoke product REST APIs using a bash-like CLI.

Git integration

Native Git integration in the shell and GUI allows you to clone, commit, manage branches, manage repositories, push, pull, fetch, and retrieve history.

Locally Managed Clones

Code checkouts from your code management repository create a local clone hosted within Codenvy. Codenvy protects, isolates, and manages the clone while also making it synchronized across the IDE, build and runtime systems.

Import GitHub Projects

Import existing projects from GitHub. Codenvy will detect the project type and configure the build and runtime where possible.

SSH Key Management

Codenvy automates the process of generating public SSH keys and uploading them to your external service provider.

Java: Package Management

For Java applications, a package management view of classes and files is provided as an alternative view.

Collaborative Development

Collaboration

Work in WebEx-style, Google docs-style, or GitHub clone mode with your workspace. Either invite one or more users into your workspace, or enable the workspace to be Factoried.

Multi-Cursor Editing

When multiple users are accessing the same file, multiple cursors track the work that each person is doing, and displays their actions to others.

Real-Time Communication

We provide a chat window that is connected to code pointers to enable shared workspace users to communicate where they are and what they are doing.

Private Workspaces

Workspaces can be public or private. Workspace owners can select the type of privacy mode and determine which users have administrative or write privileges to the workspace.

ACL Controls

We provide fine-grained controls over which users have access to which projects. Developers can see the projects they own and those they have been granted permissions to in a single view.

Factories

We make cloning a workspace as easy as GitHub makes cloning a repository. Your application, code, configuration, and runtime are all cloned together. Developers who instantiate a Factory are placed in an anonymous, temporary, and fully-functional workspace isolated from permanent projects.

Read-Only Mode

Share your workspace projects as read-only to restrict certain behaviors like creating new projects or files, building and running apps, editing code, pushing changes or deploying applications. Let others see the code without jeopardizing your workspace security.

Built-In Build Framework

Integrated Build System

Codenvy has an integrated, dependency-based maven build manager to structure compilation, libraries, unit tests, and packaging. You can configure your own build sequences and even reuse libraries from Maven Central.

Parallel Compilation

Codenvy uses multi-core compute nodes for our build clusters, enabling parallel compilation for speedy builds.

Dependency Management

We automatically download necessary dependencies and handle circular and multi-module dependencies. Add a new dependency to maven and we take care of the rest.

Remote Libraries

Your applications can use libraries with artifacts that are stored remotely, such as in Maven Central. You can also publish libraries created in Codenvy directly to external repositories.

Local Library Uploads

You can upload libraries, such as JARs, that are not available in remote repositories directly into your project.

Multi-Module Projects

Create projects that have multiple artifacts within the project. Maven multi-module projects enable multiple project structures where each artifact is built and managed separately, but under a single project umbrella.

Datasource Bindings

Bind datasources to your Codenvy projects and enjoy a secure connection to a variety of databases. We take care of configuring the datasources, inserting them into the configuration files, and testing the connections.

Works with Cloud Databases

Codenvy can connect to databases and database services running on many popular platforms.

Local Datasource

Use the local HSQL file-based database management driver for testing SQL-based apps without the use of a database.

Format Code

Syntax color highlighting, code indentation, and placement of brackets are simplified with a built-in formatter. Choose from a few different formatter presets.

Block Commenting

Add or remove comments and choose between various comment types.

Line Numbers

Instantly jump to any line of code to navigate through the files.

Full Screen Mode

No distractions and maximum focus on your code. Use a full screen mode to see just your code.

Code Auto-Completion

A productive developer needs a smart IDE. Code completion is offered across a variety of languages for variables, methods, and classes.

Restrict direct access to field variables and protect them from other code by generating getters and setters.

Java: Generate Constructors

Automatically generate constructors that contain pre-defined variables defined elsewhere in the project.

Java: Refactoring

Java: Code Folding

Code Outlines

Multiple Output Panels

Multiple Themes
